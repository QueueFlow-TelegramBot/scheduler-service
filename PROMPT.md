# Claude Code Prompt — SmartCampus Scheduling Service

## Project Overview

Build a **Scheduling Service** (also called `room-service`) for the SmartCampus Bot system — a secretary queue and room reservation system used via Telegram. This service is one part of a larger microservice architecture. My name is Tudor and I own this service.

The service manages **queue rooms** where a "Queue Creator" (secretary) creates a room, and "Queue Joiners" (students) join it. When the creator requests the next person, the service pulls from a RabbitMQ queue and notifies the next student via a RabbitMQ notification topic.

## Database Models (PostgreSQL)

### Table: `rooms`

| Column       | Type         | Notes                                      |
|--------------|--------------|---------------------------------------------|
| id           | VARCHAR(5)   | Primary key, **5 random alphanumeric chars** (e.g., "aB3xQ"). Generated server-side using `secrets.token_urlsafe` or `random.choices(string.ascii_letters + string.digits, k=5)`. Must be unique — retry on collision. |
| name         | VARCHAR(255) | Room name (e.g., "Secretary Office 1")      |
| creator_id   | VARCHAR(255) | The Telegram user ID of the creator         |
| creator_name | VARCHAR(255) | Display name of the creator                 |
| status       | VARCHAR(50)  | ENUM-like: "active", "closed". Default: "active" |
| created_at   | TIMESTAMP    | Auto-set on creation (UTC)                  |
| updated_at   | TIMESTAMP    | Auto-updated (UTC)                          |

**Room ID generation helper** — put this in `app/utils.py`:
```python
import string, random

def generate_room_id(length: int = 5) -> str:
    """Generate a short alphanumeric room ID like 'aB3xQ'."""
    alphabet = string.ascii_letters + string.digits
    return ''.join(random.choices(alphabet, k=length))
```
When creating a room, generate an ID, check for collision in DB, retry if needed (max 5 attempts, then raise 500).

### Table: `queue_entries`

| Column    | Type         | Notes                                          |
|-----------|--------------|-------------------------------------------------|
| id        | UUID         | Primary key                                     |
| room_id   | VARCHAR(5)   | FK → rooms.id                                   |
| user_id   | VARCHAR(255) | Telegram user ID of the joiner                  |
| user_name | VARCHAR(255) | Display name of the joiner                      |
| position  | INTEGER      | Position in queue (1-based, auto-incremented per room) |
| status    | VARCHAR(50)  | "waiting", "notified", "served", "cancelled"    |
| joined_at | TIMESTAMP    | Auto-set on creation (UTC)                      |

### 5. `POST /rooms/{room_id}/join`

**Summary:** Join a queue room  
**Request Body:**
```json
{
  "user_id": "987654321",
  "user_name": "Ion Studentul"
}
```
**Response 200:**
```json
{
  "room_id": "aB3xQ",
  "room_name": "Secretary Office 1",
  "creator_name": "Maria Popescu",
  "position": 4,
  "people_in_front": 3
}
```
**Logic:**
1. Validate the room exists and `status = "active"`
2. Check if the user is already in the queue with `status = "waiting"` for this room — if so, return 409 Conflict
3. Calculate position = count of existing `waiting` entries + 1
4. Insert a `queue_entries` row with `status = "waiting"`
5. **Publish to RabbitMQ** topic `room.{room_id}` with body: `{ "user_id": "987654321", "user_name": "Ion Studentul" }`
6. Return the response

**Response 404:** Room not found  
**Response 409:** User already in this queue

**Tests:**
- Join a valid room → 200, position = 1
- Join again → 409
- Join a non-existent room → 404
- Two users join → second gets position = 2
- Verify RabbitMQ publish was called with correct routing key and body (mock aio-pika)

### 6. `POST /rooms/{room_id}/next`

**Summary:** Get the next person in the queue (called by the room creator)  
**Request Body:**
```json
{
  "creator_id": "123456789"
}
```
**Response 200:**
```json
{
  "room_id": "aB3xQ",
  "room_name": "Secretary Office 1",
  "next_user_id": "987654321",
  "next_user_name": "Ion Studentul",
  "position_served": 1,
  "remaining_in_queue": 2
}
```
**Logic:**
1. Validate the room exists and the `creator_id` matches the room's creator
2. **Consume one message** from the RabbitMQ queue `room.{room_id}` (use `basic_get` — a single pull, NOT a continuous consumer)
3. If no message is available, return 204 No Content (queue is empty)
4. Parse the message to get `user_id` and `user_name`
5. Update the `queue_entries` row for this user in this room → set `status = "notified"`
6. **Publish to RabbitMQ** topic `notification.{user_id}` with body: `{ "room_id": "aB3xQ", "room_name": "Secretary Office 1" }`
7. Return the response with the next user's info

**Response 204:** No one in queue  
**Response 403:** `creator_id` does not match the room's creator  
**Response 404:** Room not found  

**Tests:**
- Create room, join with 2 users, call next → returns first user, remaining = 1
- Call next again → returns second user, remaining = 0
- Call next on empty queue → 204
- Call next with wrong creator_id → 403
- Verify notification publish was called with correct topic and body (mock aio-pika)

---

### 7. `PATCH /rooms/{room_id}/close`

**Summary:** Close a room (no more joins allowed)  
**Request Body:**
```json
{

```
**Response 200:**
```json
{

```
**Logic:**
1. Validate room exists and `creator_id` matches
2. Set `status = "closed"`
3. Update all remaining `waiting` queue entries to `cancelled`

**Response 403/404:** Same as above.

**Tests:**
- Close room successfully → 200
- Try to join a closed room → 400

## RabbitMQ Integration Details

### Connection Manager (`app/rabbitmq.py`)

- Use **aio-pika** for async RabbitMQ operations
- Manage connection and channel as app-level singletons (initialize on FastAPI lifespan startup, close on shutdown)
- Use a **topic exchange** named `smartcampus`
- Expose two methods:
  - `publish(routing_key: str, body: dict)` — publish a JSON message to the exchange
  - `pull_one(queue_name: str) -> Optional[dict]` — consume a single message from a queue using `basic_get` (non-blocking pull)
- On first publish to `room.{room_id}`, the queue `room.{room_id}` should be declared if it doesn't exist (durable queue, bound to the exchange with routing key `room.{room_id}`)
- On first publish to `notification.{user_id}`, the queue `notification.{user_id}` should be declared if it doesn't exist (auto-delete queue, bound with routing key `notification.{user_id}`)

### Message Schemas

**room.{room_id}** (produced when user joins):
```json
{
  "event": "user_joined",
  "user_id": "987654321",
  "user_name": "Ion Studentul",
  "room_id": "aB3xQ",
  "timestamp": "2025-01-01T10:05:00Z"
}
```

**notification.{user_id}** (produced when creator calls next):
```json
{
  "event": "your_turn",
  "user_id": "987654321",
  "room_id": "aB3xQ",
  "room_name": "Secretary Office 1",
  "timestamp": "2025-01-01T10:10:00Z"
}
```

## Structured Logging

- Use Python's `logging` module configured with **JSON output** (use `python-json-logger`)
- Every log line must include: `timestamp`, `level`, `service` ("scheduling-service"), `message`
- Every request log should include: `method`, `path`, `status_code`, `duration_ms`
- RabbitMQ operations should log: `action` ("publish" / "consume"), `routing_key`, `queue_name`, `success` (bool)
- Use FastAPI middleware for request/response logging
- Log levels: DEBUG for internal flow, INFO for business events (room created, user joined, next called), WARNING for edge cases (empty queue), ERROR for failures

## Configuration (`app/config.py`)

Use `pydantic-settings` with `BaseSettings`. **Priority order (highest → lowest):**
1. **Environment variables** (set in shell or CI/CD)
2. **`.env` file** (local dev convenience)
3. **Default values** (hardcoded fallbacks in the class)

This is pydantic-settings' default behavior — just make sure `env_file = ".env"` is set in `model_config`.

```python
from pydantic_settings import BaseSettings, SettingsConfigDict

class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
        extra="ignore",            # ignore unknown env vars
    )

    # Database
    DATABASE_URL: str = "postgresql+asyncpg://postgres:postgres@localhost:5432/smartcampus"
    
    # RabbitMQ
    RABBITMQ_URL: str = "amqp://guest:guest@localhost:5672/"
    RABBITMQ_EXCHANGE: str = "smartcampus"
    
    # App
    APP_HOST: str = "0.0.0.0"
    APP_PORT: int = 8001
    LOG_LEVEL: str = "INFO"
    
    # Room ID
    ROOM_ID_LENGTH: int = 5
```

Instantiate as a singleton: `settings = Settings()` at module level, import everywhere.

The `.env.example` file should list all variables with their defaults and comments explaining each one.
